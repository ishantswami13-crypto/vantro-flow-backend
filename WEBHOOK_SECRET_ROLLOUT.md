# Webhook Secret Rollout

Status: required before real payment, WhatsApp, or voice automation traffic.

## Railway Variables

Set these in Railway production variables. Do not commit their values.

- `RAZORPAY_WEBHOOK_SECRET`
- `VOICE_WEBHOOK_SECRET`
- `WHATSAPP_WEBHOOK_SECRET`
- `REQUIRE_SIGNED_PUBLIC_BILLS=true` after signed public links are generated by the app
- `PUBLIC_LINK_SECRET` for public bill link signing, or keep `JWT_SECRET` as temporary fallback
- `ENABLE_AUTH_COOKIES=true` after frontend cookie rollout is tested
- `COOKIE_AUTH_SECURE=true`
- `COOKIE_AUTH_SAMESITE=None` while frontend and backend are on different production domains

## Provider Configuration

- Razorpay: configure the webhook endpoint as `/api/payments/webhook` and set the same webhook secret in Railway.
- Voice/Twilio: include `x-vantro-webhook-secret` or a provider-native validated signature before enabling production call flows.
- WhatsApp/WATI/Interakt: include `x-vantro-webhook-secret` or provider-native signature verification before enabling production inbound processing.

## Verification

Run after deployment:

```powershell
$base = "https://vantro-flow-backend-production.up.railway.app"
Invoke-WebRequest "$base/api/auth/me" -Headers @{ Authorization = "Bearer invalid" } -SkipHttpErrorCheck
Invoke-WebRequest "$base/api/payments/webhook" -Method Post -ContentType "application/json" -Body "{}" -SkipHttpErrorCheck
Invoke-WebRequest "$base/api/voice/status" -Method Post -ContentType "application/json" -Body "{}" -SkipHttpErrorCheck
```

Expected:

- Invalid auth token returns `401`.
- Unsigned Razorpay webhook returns `400`.
- Unsigned voice webhook returns `403`.

## Rollback

If a provider cannot send secrets yet, pause that integration instead of allowing unsigned production webhooks.
